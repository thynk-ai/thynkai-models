# Security policy

Please do not report security issues in public issues or pull requests.

## How to report

Send a private report to the maintainers including:
- description and impact
- affected entries (model ids / versions) if applicable
- reproduction details (if relevant)

## What counts as a security issue here

- integrity issues (tampered artifacts, mismatched digests)
- malicious links or supply-chain style attacks in registry references
- CI/workflow vulnerabilities
